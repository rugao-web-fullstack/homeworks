一、Web URL
	1、用户登录界面
		url:user/login
		作用：用户登录界面

	2、用户注册界面
		url:user/register
		作用：用户注册界面

	3、用户主界面
		url:user/main
		作用：登录后用户主界面，可跳转邮件列表界面，写邮件界面，登出

	4、邮件列表界面
		url:email/list
		作用：显示用户收到的邮件列表，可跳转到具体某页面读邮件

	5、读邮件界面
		url:email/read
		作用：用户读取某邮件

	6、写邮件界面
		url:email/write
		作用：用户写邮件并发送

二、Web API
	1、读取用户列表
		url: /users
		method: GET
		作用: 读取用户列表

	2、读取单个用户
		url: /users
		method: GET
		data: id=1
		作用: 读取单个用户
	
	3、注册
		url: /users
		method: POST
		data: action=register
		作用: 返回注册信息到数据库

	4、登录
		url: /users
		method: POST
		data: action=login
		作用: 返回登录信息并与数据库信息比较

	5、查看邮件列表
		url: /emails
		method: GET
		作用: 读取邮件信息
	
	6、查看单个邮件
		url: /emails
		method: GET
		data: id=1
		作用: 读取单个邮件信息

	7、发邮件
		url: /emails
		method: POST
		data: action=send
		作用: 返回邮件信息到数据库
